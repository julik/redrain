# ZAY-433 — Ruby port of `rain-sdk` (python) → `redrain`

## Context

We're evaluating [Rain](https://raincards.xyz) for card issuing. To integrate from Ruby we
either scrape their API docs or port their SDK. Porting wins.

Rain ships [`rain-sdk-python`](https://github.com/SignifyHQ/rain-sdk-python) — a
Stainless-generated client over their *Issuing API v1.2.1*: **60 endpoints, 48 paths,
~35k LOC** of which the vast majority is Stainless ceremony (httpx base client, pydantic
models, async mirrors, raw/streaming response wrappers, `_transform`/`_qs`/`_compat`).

We did this once before for Amplitude → [`julik/ramplitude`](https://github.com/julik/ramplitude).
Same workflow here, same conventions (see `tmp/ramplitude/CLAUDE.md` and
`tmp/ramplitude/llm/plans/port-plan.md`).

**Outcome:** a `redrain` gem with a stdlib-only runtime, the Python SDK's public call
shape preserved (`client.users.create`, `client.cards.pin.update`,
`client.applications.company.ubo.document.upload`), typed snake_case response models, and
a committed code generator so we can re-sync when Rain bumps the spec.

### Reference material already cloned

| Path | What |
| --- | --- |

| `tmp/ramplitude/` | Our previous port — conventions to mirror. |

### Facts established from the spec + SDK

- **Auth:** `Api-Key: <key>` header (`components.securitySchemes.ApiKeyAuth`). Not Bearer.
- **Environments:** `dev` → `https://api-dev.raincards.xyz/v1/issuing`,
  `production` → `https://api.raincards.xyz/v1/issuing`. Python defaults to **dev**.
- **Env vars:** `RAIN_API_KEY`, `RAIN_BASE_URL`.
- **Wire format is camelCase** (`firstName`, `walletAddress`, `companyId`); Python maps to
  snake_case via pydantic aliases. We do the same.
- **Content types:** requests `application/json` ×24, `multipart/form-data` ×6 (document /
  evidence / receipt uploads). Responses `application/json` ×47,
  `application/octet-stream` ×2 (dispute evidence download, transaction receipt download).
- **Status codes:** 200/201/202 with bodies, **204 ×11** with no body → return `nil`.
  Error responses (400/401/403/404/500) have **no schema** — bodies are opaque, parse
  best-effort.
- **Only 16 named component schemas**; every request body and most response bodies are
  inlined. The generator must synthesise names for inline shapes exactly the way Stainless
  did (`CompanyRetrieveResponse`, `UserListResponse`, …) so our types line up with `api.md`.
- **Pagination is cursor-based and manual** — `cursor` + `limit` query params, list
  endpoints return bare JSON arrays. No envelope, no auto-pager in Python.
- **Retries:** Python defaults to 2, exponential 0.5s → 8s cap, 60s timeout / 5s connect.

### Explicitly dropped from the port

Async client, streaming, `with_raw_response` / `with_streaming_response` wrappers,
`_strict_response_validation`, `RAIN_CUSTOM_HEADERS` parsing, `_qs`/`_compat`/`_transform`.
None have a Ruby analogue worth the code.

## Design

### Naming

Gem `redrain`, module `Redrain` — mirrors the `ramplitude` trick and keeps us off the
generic `Rain` constant in a Rails app.

```ruby
rain = Redrain::Client.new(api_key: ENV.fetch("RAIN_API_KEY"), environment: :dev)

user = rain.users.create(email: "a@b.com", first_name: "A", last_name: "B")
user.id                                  # => "usr_..."
user.phone_country_code                  # snake_case readers, camelCase on the wire

rain.cards.list(user_id: user.id, status: "active")
rain.cards.pin.update(card_id, encrypted_pin_block: "...")
rain.applications.company.ubo.document.upload(ubo_id, company_id: cid,
                                              document: File.open("id.png"), type: "passport")
rain.transactions.receipt.retrieve(txn_id)   # => String (octet-stream)
```

### Generated vs. hand-written

`rake generate` reads `openapi/rain-issuing.json` + `dev/resource_map.yml` and rewrites
`lib/redrain/models/` and `lib/redrain/resources/` (plus per-endpoint smoke tests). Output
is **committed** and carries a `# Generated by `rake generate` — do not edit` banner.
Everything else is hand-written and the generator never touches it.

`dev/resource_map.yml` is the hand-maintained piece: `operationId` → resource path +
method name, transcribed from `tmp/rain-sdk-python/api.md`. This is the part the spec
can't tell us (Stainless keeps it in its own config), and it's what keeps our call sites
identical to Python's.

### Core hand-written pieces

**`Redrain::Model`** — hash-backed, declarative:

```ruby
class IssuingUser < Redrain::Model
  field :id,                :string
  field :first_name,        :string, api: "firstName", required: true
  field :address,           PhysicalAddress
  field :is_active,         :boolean, api: "isActive"
  field :created_at,        :time,    api: "createdAt"
end
```

- Unknown keys from the API are **preserved** in `#raw` and reachable via `#[]` — Rain
  adding a field must never blow up a running integration.
- `.from_api(hash)` for parsing, `#to_h` emits wire format, `#inspect` shows snake_case.
- Coercers: `:string :integer :number :boolean :time :hash`, nested `Model` subclasses,
  and arrays thereof. Enums documented in a comment + exposed as a frozen constant, but
  **not** validated on read (spec enums drift; a rejected read is worse than a surprising one).

**`Redrain::HTTPClient`** — modelled on `tmp/ramplitude/lib/ramplitude/http_client.rb`
(`Net::HTTP`, explicit `Accept-Encoding` + manual gzip decode, timeouts from constants).
Additions this port needs:

- Retry on 408/409/429/5xx and `Net::OpenTimeout`/`ReadTimeout`/`EOFError`/`ECONNRESET`,
  max 2 by default, `0.5 * 2**n` capped at 8s with jitter, honouring `Retry-After`.
- `multipart/form-data` encoding for the 6 upload endpoints, driven by `Redrain::Upload`.
- Raises the typed error tree on non-2xx instead of returning a Response object (this is a
  request/response API, unlike Amplitude's fire-and-forget batching).

**`Redrain::Errors`** — mirrors `_exceptions.py`:
`Error` < `StandardError`; `APIError`; `APIConnectionError` / `APITimeoutError`;
`APIStatusError` carrying `status`, `body`, `request_id`, with subclasses
`BadRequestError` (400), `AuthenticationError` (401), `PermissionDeniedError` (403),
`NotFoundError` (404), `ConflictError` (409), `UnprocessableEntityError` (422),
`RateLimitError` (429), `InternalServerError` (5xx).

**`Redrain::Page`** — the one place we improve on Python. List endpoints take
`cursor:`/`limit:`; we add `auto_paging_each` so callers stop hand-rolling cursor loops:

```ruby
rain.transactions.auto_paging_each(user_id: uid) { |txn| ... }   # Enumerator without a block
```

### Generator behaviour (`dev/generate.rb`)

1. Load spec + `resource_map.yml`; fail loudly on any operationId present in one and not
   the other — that's how we catch a Rain spec bump adding endpoints.
2. Emit a model class per named component schema, plus one per inline request/response
   shape, named per `api.md` so the mapping stays legible.
3. Emit a resource class per node of the map, with a method per operation:
   path params positional, body/query params keyword, `nil` return for 204,
   `String` return for octet-stream, model return otherwise.
4. Carry the spec's `description` into RDoc comments — Rain's descriptions are good and
   this is where the docs live.
5. Emit `test/resources/<resource>_test.rb` WebMock smoke tests asserting method, path,
   `Api-Key` header, camelCase body keys and parsed return type for every endpoint.

### Testing

Minitest + WebMock (`tmp/ramplitude/test/test_helper.rb` conventions). Hand-written unit
tests for `model.rb` (mapping, unknown-key passthrough, nested/array coercion, nil
handling), `http_client.rb` (retry/backoff, `Retry-After`, multipart body, gzip, each
error subclass), `client.rb` (env-var fallback, environment→base_url resolution, ambiguous
`RAIN_BASE_URL` + `environment:` raising as Python does), `page.rb`. Generated smoke tests
cover all 60 endpoints. Integration tests under `test/integration/` gated on
`RAIN_API_KEY` being set, mirroring `tmp/rain-sdk-python/tests/custom/integration/`.
CI matrix Ruby 3.1–3.4, copied from ramplitude's `.github/workflows/test.yml`.

## Steps

1. **Skeleton** — gemspec, Gemfile, Rakefile, CI, LICENSE, `CLAUDE.md`, `.ruby-version`.
   Vendor `tmp/rain-openapi.yml` → `openapi/rain-issuing.json` (pretty-printed) and
   `openapi/stats.yml`. Check this plan in at `llm/plans/port-plan.md`.
2. **Errors + HTTP client** — `errors.rb`, `http_client.rb`, `upload.rb`. Tests first;
   this is the layer everything else trusts.
3. **Model base** — `model.rb` field DSL + coercion + unknown-key passthrough, with tests.
4. **Client + resource base** — `client.rb`, `resource.rb`, `page.rb`. Environment
   resolution, lazy memoised resource accessors.
5. **`dev/resource_map.yml`** — transcribe all 60 rows from `tmp/rain-sdk-python/api.md`.
6. **`dev/generate.rb` + `rake generate`** — emit models, then resources, then smoke
   tests. Iterate until output is clean Ruby that reads like it was written by hand.
7. **Run the generator, review the output**, commit it. Fix generator bugs at the
   generator, never in the output.
8. **Full test pass** — `rake test` green on 3.1–3.4.
9. **README + `llms.txt`** — install, auth, environments, per-resource examples, error
   handling, pagination, uploads, "behaviours an agent should NOT change". Mirror
   `tmp/ramplitude/llms.txt` structure; register `llms_txt_uri` in gemspec metadata.
10. **`rake sync_spec`** — re-fetch the spec URL from `openapi/stats.yml`, diff against the
    vendored copy, report drift. The re-sync story for future Rain versions.
11. **Smoke against Rain dev** — run `test/integration/` with a real dev key.

## Verification

- `rake test` — unit + generated smoke tests, all green, on Ruby 3.1 through 3.4.
- `rake generate` is **idempotent**: running it on a clean tree leaves `git status` empty.
  This is the check that the committed output really came from the spec.
- `ruby -Ilib -e 'require "redrain"'` with no bundle — proves zero runtime deps.
- Endpoint coverage assertion: a test that walks `dev/resource_map.yml` and asserts every
  one of the 60 operationIds resolves to a real method on a real resource object, so
  nothing silently goes missing.
- Live smoke against `environment: :dev` with `RAIN_API_KEY` set: create a user
  application, retrieve it, list cards, fetch balances, upload a document (multipart),
  download a receipt (octet-stream). Both binary and multipart paths exercised for real —
  WebMock can't prove those encode correctly.
- Manual read-through of the generated `users.rb` and `applications/company/ubo/` against
  `api.md` to confirm signatures and nesting match Python call-for-call.
